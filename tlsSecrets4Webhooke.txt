 Secrets for Webhooks
# Creating TLS Secret
kubectl create secret tls <secret-name> \
  --cert=<path-to-cert-file> \
  --key=<path-to-key-file> \
  --namespace=<namespace>

# Example:
kubectl create secret tls webhook-server-cert \
  --cert=server.crt \
  --key=server.key \
  --namespace=webhook-demo

# Notes:
- cert = public certificate (.crt/.pem)
- key = private key (.key)
- cert and key must match
- Secret lives in namespace where webhook deployment references it

5. Quick Tip: Generating Self-Signed Certs (for testing webhooks)
openssl req -newkey rsa:2048 -nodes -keyout server.key \
  -x509 -days 365 -out server.crt -subj "/CN=webhook-demo.default.svc"

# Use generated server.crt and server.key to create TLS secret

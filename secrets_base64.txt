Option 1: Individual env vars (valueFrom.secretKeyRef)
env:
  - name: DB_USER
    valueFrom:
      secretKeyRef:
        name: app-secret
        key: DB_USER
  - name: DB_PASS
    valueFrom:
      secretKeyRef:
        name: app-secret
        key: DB_PASS
  - name: API_KEY
    valueFrom:
      secretKeyRef:
        name: app-secret
        key: API_KEY
  - name: TOKEN
    valueFrom:
      secretKeyRef:
        name: app-secret
        key: TOKEN
  - name: SERVICE_URL
    valueFrom:
      secretKeyRef:
        name: app-secret
        key: SERVICE_URL


✅ Pattern you can memorize:

name: → env var name

valueFrom: → source

secretKeyRef: → type = Secret

name: → Secret name

key: → key inside Secret

Option 2: Inject all keys with envFrom
envFrom:
  - secretRef:
      name: app-secret


No need to list 5 pairs individually

All Secret keys become env vars automatically:

DB_USER=mysql
DB_PASS=pass
API_KEY=123456
TOKEN=token123
SERVICE_URL=http://localhost

✅ Quick tip for exam

If they give you multiple keys, use envFrom unless they specifically ask for only 1–2 keys.

The YAML structure for valueFrom.secretKeyRef is always the same, just copy-paste the pattern 5 times.
